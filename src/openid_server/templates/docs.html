Authentication

Endpoint: {{ url_for('frontend.auth', _external=True) }}
Method: GET
Parameter Description Required?
client_id Client ID of your app required
response_type Response type. Only code is supported (Authorization Code Flow) required
redirect_uri Redirect URI. One of the URIs registered with the client required
scope Space separated list of scopes. One or more of: openid, email, profile
Should include openid for OpenID Connect. email and profile are optional. 		required
state Random string, generated by the client. required
response_mode How the code is passed to the redirect URI. One of: query (default), fragment, form_post optional
code_challenge PKCE Code challenge required for public clients. Optional (but recommended) for private clients.
code_challenge_method Method used for the code challenge. Only S256 is supported and it's the default. optional
nonce Random generated string inserted into the generated JWT. optional
prompt One of none, login, consent
Controls the authentication/consent UI (e.g. 'none' for silent renew).
If absent, server chooses automatically the best prompt if needed. 		optional

If successful, user will be prompted to Allow or Deny access to their data according to the scope.
The user will then be redirected to the provided redirect_uri with the following parameters in either the query string, fragment for form post body, depending on the response_mode.

Parameter Description Present?
code The code to be used in the Token endpoint. The code is valid for about 5 minutes. always
state The same state as passed in to the Authorization endpoint. always
iss The issuer representing this server.
Currently: {{ settings.issuer }} 		always

Alternatively, if the user presses Deny or the provided prompt could not be performed (e.g. could not continue with prompt=none due to missing user session), the user will be redirected to the redirect_uri in the same manner with an error instead

Parameter Description Present?
error The error name. access_denied if the user denied the access, or one of login_required or consent_required if the specified prompt failed. always
error_description Human-readable description of the error. always
state The same state as passed in to the Authorization endpoint. always
iss The issuer representing this server.
Currently: {{ settings.issuer }} 		always

Token

Endpoint: {{ url_for('api.token', _external=True) }}
Method: POST
Parameter Description Required?
client_id Client ID of your app required
client_secret Client Secret of your app required for private clients. Do not use with public clients.
code The code received through the redirect. required
grant_type Must be set to authorization_code. required
redirect_uri Redirect URI. One of the URIs registered with the client. The same that was used in the Authorization step. required
code_verifier The second part of the PKCE Code Challenge.
If a code_challenge was provided in the Authorization request, then the code_verifier must be sent here. 		required for public clients. Optional (but recommended) for private clients.

Response if successful: JSON encoded payload

Parameter Description Present?
access_token JWT Token to access ther userinfo endpoint. always
token_type Token type of the Access Token. Always "Bearer" always
id_token JWT OpenID Connect token. only if scope included openid
expires_in The lifetime of both tokens.
Note: the token are JWT and contain an authoritative "exp" field themselves and the value returned here is more of a hint. 		always

The claims in the id_token JWT are as follows

Claim Description Present?
Standard JWT claims
sub The subject containing user UUID always
aud Audience containing the Client ID only if scope included openid
iat Timestamp when the token was created. always
exp Timestamp when the token expires. always
iss Issuer. Currently always {{ settings.issuer }} always
jti Random string to prevent replay attacks always
Authorization related claims
scope The scopes the user accepted always
nonce Nonce value from the authorization request if provided only if present in authorization request
auth_time The timestamp of when the user gave authorization always
at_hash Access Token Hash. Left half of the sha256 hash of the access_token (mix-up attack) always
User Claims
email The users email. only if scope included email
email_verified A boolean indicating if the users email is verified. only if scope included email
name The users display name only if scope included profile
given_name The users given name / first name only if scope included profile and non-empty
family_name The users family name / last name only if scope included profile and non-empty
picture Public URL to a profile picture only if scope included profile and non-empty

UserInfo

Endpoint: {{ url_for('api.userinfo', _external=True) }}
Method: GET

Requires access_token in Authorization header:
Authorization: Bearer your_access_token

Parameter Description Required?

No parameters required

Response if successful: JSON encoded payload.
The fields are identical to the corresponding JWT claims in the id_token.

Claim Description Present?
sub The subject containing user UUID always
email The users email. only if scope included email
email_verified A boolean indicating if the users email is verified. only if scope included email
name The users display name only if scope included profile
given_name The users given name / first name only if scope included profile and non-empty
family_name The users family name / last name only if scope included profile and non-empty
picture Public URL to a profile picture only if scope included profile and non-empty
